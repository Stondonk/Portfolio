Games & Projects

List and description of all my major Projects

Tank Tracks a game created in Unity is one of the more recent better performing games I've made created for one of the many LOWREZJAMs I've been a part of, this one being LOWREZJAM 2021. The game was created not only around the theme of the low resolution but also around the idea of an easily understandable gameplay mechanic so I looked at other games from the Kirby series for inspiration. I then used this to develop an idea of using Kirby's inhale ability to retrieve dodgeballs being shot by enemies and redirecting them back at toward the enemy just like how Kirby can inhale rockets and fire them back out at opposing obstacles.

The main mechanic of this game is the inhaling and then the further collection / redirection of the enemy projectiles back at them to damage or possibly even kill them. The game only has a handful of levels each feeling unique in its design, using different environments and similar setups to give the player a feeling of progression without overdoing it to make an overall bit sized yet fun experience.

Flip Tennis was the first ever major game release for me back in the summer of 2021 and ever since has been one of my favorite projects to have worked on. Created for the simplistic nature of phones and mobile devices the game has a basic control scheme and gameplay loop with many collectables and unlockables to be found throughout with many different game modes and ideas sprinkled within.

The game unlike most wasn't created for a game jam but for the soul reason to make a fun and simple mobile game that could be enjoyed by all without an advert popping up every minute or so bringing misery and despair, pulling you right out of the experience with it. The whole game had many plans for expansion and growth but unfortunately the files for the game went corrupt and all the future content for the game was lost alongside it. The gameplay itself boils down to tap left and right to move and hitting a ping pong ball as fast as you can before it fades into nothing increasing the score with every swing from the player. The game has 2 player, an infinite mode and a campaign / adventure mode where you travel to various stages and complete highscore objects to progress.

New Moves is one of my more successful games made for the Juice Jam, a game jam with the objective of making the most satisfactory game imaginable in a short amount of time that being about a week or so, decently long for a game jam but not enough for the game I was attempting to make.

The game was made in Unity and has 4 player versus with computer controlled characters as well this game is probably one of my more enjoyable games to come back to as the functionality and gameplay still hold up quite well. As previously stated the objective for this jam was to make a very satisfying game, with that in mind I used a lot of screen shake and meaty feeling sound design to amplify the atmosphere and satisfaction factor. Another part of this game designed differently from my other projects is the artstyle now it is true that a lot of my project designs and artstyles have had a simplistic nature to them in previous times but this one was more deserving. I felt to help aid the screenshake making it harder to see the screen my artstyle should be easier to perceive so simple colors and tones were used to ensure easy player visibility even in the worst of times so each player, computer and environment have differing and strongly contracting colors used to help make the overall experience clearer and more pleasant to view.

Mighty Versus is a project made over the duration of two weeks created for a Game Jam called LOWREZJAM 2022 the theme and objective being to create a game within a 64 pixel by 64 pixel resolution. For this challenge I decided to make a recreation / Demake of a new and very popular game at the time MultiVersus. But my challenge didn't just lay in the low resolution but also in the program I had decided to make the game in, or the lack there of, because this project was written entirely within javaScript with no external tools aiding in its creation.

This Project was the main point of development for my javascript knowledge base which would go on to not only aid me in the development of the website your currently on, but also helped strengthen my understanding of higher level programming languages. Which may seem useless to transition from a lower end language but to something higher spec but each language and level of Programming have their own quirks, something very important that I learnt throughout this games production.

Programming Stuff

These are some the Larger Programming projects I've worked on

Flip Tennis was one of the projects I kept coming back to. So During a game jam called the Floppy disk game jam I was challenged to make a game within a small file size ranged between 3 categories allotted, those being 80kb, 256kb and 1.4mbs I chose to aim for the hardest that being 80kb and so I would have to tear every little bit of data I could to ensure a smaller size.

This project was created using only the programming language C and a popular renderer called OpenGL a single dll file used to assist in rendering for a .exe file that runs the application all together totaling 264kb which when placed in a rar file converts into about 89 kilobytes of data an incredibly small impressive amount for any game and so that's where the name comes from. Finally the last tidbit of relevant info about this game is its construction over an entire script rather than being made though a selection of separate scripts and bodies simplifying the data needed for the game once compiled. This becomes incredibly useful to know if I ever want to port this application to another other language or software. For instance there's a game called retro gadgets where you build and program little devices you make with lua code. I recreated a version of Flip tennis in that game by using the code made in that single script, showing the possibilities that this game might have for testing and showcasing in software for the future.

After creating the low level language game of flip tennis 89k I wished to make a better more improved engine and tools culminating in a project that would show off the improvements and capabilities of the engine. That project evolved into The Top Down Shooter, depicted here. The Top Down shooter has many aspects to it a lot of that being more engine based stuff which I've placed in a separate tab called C++ engine above, But for the most part the two are interlinked developed alongside one another which I feel deserve separate tabs to accommodate.

The project has two gamemodes a battle arena mode spawning in random enemies across an arena dueling for survival, the second mode is an endless mode both modes are endless but its used as a simple term for how the player gets set a random room and has to finish said room by killing all the enemies before they inevitably kill him. This game is intentionally very high score based, hopefully in the future being developed out into a more robust roguelike that feels as if more is going on. Finally this project is one of my more advanced bar bones games those being projects created without engines and heavy restrictions in place so Hopefully once I feel its finished development ,it should feel as if those restrictions didn't mold the game and more so that the game was made for its own sake.

My C++ game engine came around as a consequence to the C++ shooter game I developed and was created with the soul purpose of aiding and developing the gameplay abilities of what programming in C++ could do for my projects in the future.

This engine relies heavily on the object oriented side of C++ functions turning every thing bar editing and debug tools into a object to easily be spawned in and created at a whim for solo functionality. A lot of these solo Objects revolve around a subclass called gameobject but to explain what a subclass is, I first need to explain A class. A class is an object that can be stored as a variable type with its own separate functions and child variables within and a subclass is when you take that class and make essentially a copy of all its properties, functions and variables and append other assets such as more functions and variables to it under a different title. Using this methodology I have created a workflow that mirrors some of the functionalities of other big name engines such as Unity or Unreal with it placement and object based system, software I have also used in the past to create projects.
