My C++ game engine came around as a consiquence to the C++ shooter game I developed and was created with the soul purpose of aiding and developing the gameplay abilities of what programming in C++ could do for my projects in the future.

This engine relies on heavily on the object oriented side of C++ functions turning every thing bar editing and debug tools into a object to easily be spawned in and created at a whim for solo functionality. A lot of these solo Objects revolve around a subclass called gameobject but to explain what a subclass is, I first need to explain A class. A class is an object that can be stored as a variable type with its own sperate functions and child varaibles within and a subclass is when you take that class and make essentailly a copy of all its propties, functions and variables and append other assets such as more functions and variables to it. Using this methodology I have created a workflow that mirrors some of the functionalities of other big name engines such as UNITY or UNREAL software I have also used in the past the create projects.
